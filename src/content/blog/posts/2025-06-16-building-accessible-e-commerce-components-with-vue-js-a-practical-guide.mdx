<p>As frontend developers, we often focus on making our e-commerce applications visually appealing and functionally robust. However, accessibility is equally crucial—not just for compliance, but for creating inclusive experiences that serve all users. In this post, I'll share practical strategies for building accessible e-commerce components using Vue.js, drawing from real-world implementation experience.</p>

<h2>Why Accessibility Matters in E-commerce</h2>

<p>E-commerce accessibility isn't just about doing the right thing (though it absolutely is). It's also about business impact:</p>

<ul>
<li><strong>Market Reach</strong>: Over 1 billion people worldwide have disabilities</li>
<li><strong>SEO Benefits</strong>: Many accessibility practices improve search engine rankings</li>
<li><strong>Legal Compliance</strong>: Accessibility standards are increasingly required by law</li>
<li><strong>Better UX for Everyone</strong>: Accessible design often improves usability for all users</li>
</ul>

<h2>Common Accessibility Pitfalls in E-commerce</h2>

<p>Before diving into solutions, let's identify the most frequent accessibility issues I've encountered in e-commerce applications:</p>

<ol>
<li>Missing or inadequate form labels</li>
<li>Poor keyboard navigation support</li>
<li>Insufficient color contrast</li>
<li>Missing alternative text for product images</li>
<li>Inaccessible modal dialogs and dropdowns</li>
</ol>

<h2>Building an Accessible Product Card Component</h2>

<p>Let's start with a fundamental e-commerce component—the product card. Here's how to implement it with accessibility in mind:</p>

<pre><code class="language-vue">&lt;template&gt;
  &lt;article 
    class="product-card" 
    :aria-labelledby="`product-title-${product.id}`"
    role="article"
  &gt;
    &lt;!-- Product Image with Proper Alt Text --&gt;
    &lt;div class="product-image-container"&gt;
      &lt;img 
        :src="product.image" 
        :alt="product.imageAlt || `${product.name} - ${product.category}`"
        class="product-image"
        loading="lazy"
      /&gt;
      &lt;button 
        v-if="product.hasQuickView"
        @click="openQuickView"
        class="quick-view-btn"
        :aria-label="`Quick view ${product.name}`"
      &gt;
        &lt;EyeIcon aria-hidden="true" /&gt;
      &lt;/button&gt;
    &lt;/div&gt;

    &lt;!-- Product Information --&gt;
    &lt;div class="product-info"&gt;
      &lt;h3 
        :id="`product-title-${product.id}`"
        class="product-title"
      &gt;
        &lt;router-link 
          :to="`/products/${product.slug}`"
          :aria-describedby="`product-price-${product.id}`"
        &gt;
          {{ product.name }}
        &lt;/router-link&gt;
      &lt;/h3&gt;

      &lt;div 
        :id="`product-price-${product.id}`"
        class="product-price"
        :aria-label="`Price: ${formatPrice(product.price)}`"
      &gt;
        &lt;span class="sr-only"&gt;Price:&lt;/span&gt;
        &lt;span class="current-price"&gt;{{ formatPrice(product.price) }}&lt;/span&gt;
        &lt;span 
          v-if="product.originalPrice" 
          class="original-price"
          :aria-label="`Original price: ${formatPrice(product.originalPrice)}`"
        &gt;
          {{ formatPrice(product.originalPrice) }}
        &lt;/span&gt;
      &lt;/div&gt;

      &lt;!-- Add to Cart Button --&gt;
      &lt;button 
        @click="addToCart"
        class="add-to-cart-btn"
        :disabled="!product.inStock"
        :aria-label="`Add ${product.name} to cart`"
        :aria-describedby="!product.inStock ? `out-of-stock-${product.id}` : null"
      &gt;
        {{ product.inStock ? 'Add to Cart' : 'Out of Stock' }}
      &lt;/button&gt;

      &lt;div 
        v-if="!product.inStock"
        :id="`out-of-stock-${product.id}`"
        class="sr-only"
      &gt;
        This item is currently out of stock
      &lt;/div&gt;
    &lt;/div&gt;
  &lt;/article&gt;
&lt;/template&gt;</code></pre>

<h2>Key Accessibility Features Implemented</h2>

<h3>1. Semantic HTML Structure</h3>
<ul>
<li>Used <code>&lt;article&gt;</code> for the product card container</li>
<li>Proper heading hierarchy with <code>&lt;h3&gt;</code> for product titles</li>
<li>Meaningful link text and button labels</li>
</ul>

<h3>2. ARIA Labels and Descriptions</h3>
<ul>
<li><code>aria-labelledby</code> connects the product card to its title</li>
<li><code>aria-describedby</code> provides additional context for prices and stock status</li>
<li><code>aria-label</code> offers descriptive text for interactive elements</li>
</ul>

<h3>3. Keyboard Navigation Support</h3>
<ul>
<li>All interactive elements are focusable</li>
<li>Clear focus indicators with outline styles</li>
<li>Logical tab order throughout the component</li>
</ul>

<h3>4. Screen Reader Considerations</h3>
<ul>
<li>Hidden descriptive text using <code>.sr-only</code> class</li>
<li><code>aria-hidden="true"</code> for decorative icons</li>
<li>Proper price formatting with currency information</li>
</ul>

<h3>Manual Testing Checklist</h3>
<ul>
<li>Navigate using only keyboard (Tab, Enter, Space, Arrow keys)</li>
<li>Test with screen reader (NVDA, JAWS, or VoiceOver)</li>
<li>Verify color contrast ratios meet WCAG standards</li>
<li>Check that all interactive elements have focus indicators</li>
<li>Ensure form labels are properly associated</li>
</ul>

    if ('IntersectionObserver' in window) {
      const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
          if (entry.isIntersecting) {
            const img = entry.target
            img.src = img.dataset.src
            img.alt = img.dataset.alt
            observer.unobserve(img)
          }
        })
      })

      }
